# Hysteria 2 Docker: Полное руководство (2026)

Минималистичный шаблон для развертывания сервера Hysteria 2 в Docker. Оптимизирован для работы с ядрами Xray-core и sing-box.

---

## Структура проекта

```text
hysteria2-docker/
├── config.yaml          # Основной файл конфигурации сервера
├── docker-compose.yaml  # Описание сервиса и монтируемых томов
├── generate_uri.py      # Скрипт для создания ссылок подключения
├── README.md            # Данное руководство
└── hysteria/
    ├── certs/           # Директория для ручных сертификатов
    └── data/            # Директория для кеша ACME (Let's Encrypt)
```

## Сценарии развертывания (Пошаговые инструкции)

Выберите подходящий сценарий и следуйте шагам по порядку.

### 🟢 Сценарий 1: Обфускация + ACME на Домен (Рекомендуется)
Идеальный вариант для 2026 года. Требует домен (можно бесплатный), скрывает трафик под шум (Salamander), автоматически получает валидный сертификат. **Полностью совместим с Xray-core и sing-box.**

**Чеклист настройки:**
1. **Подготовка:**
   - Направьте ваш домен (A-запись) на IP-адрес сервера.
   - Убедитесь, что на сервере открыт порт **80** (нужен для ACME).
2. **Настройка config.yaml:**
   - Раскомментируйте блок `acme:` (Вариант А).
   - Впишите свой **домен** вместо `"ВАШ_ДОМЕН"`.
   - Укажите свой email.
   - Убедитесь, что блок `tls:` закомментирован.
   - Убедитесь, что включен блок `obfs: type: salamander` и задан пароль.
3. **Права доступа (только Linux):** Выполните `sudo chown -R 1000:1000 hysteria/data`.
4. **Запуск:** Выполните `docker-compose up -d`. Проверьте логи `docker logs -f hysteria2`.
5. **Подключение клиента:** Запустите `python generate_uri.py` или соберите ссылку:
   `hysteria2://user:pass@ВАШ_ДОМЕН:4433/?sni=ВАШ_ДОМЕН&obfs=salamander&obfs-password=ВАШ_ПАРОЛЬ#Hysteria_ACME`

### 🟡 Сценарий 2: Обфускация + Самоподписной сертификат
Вариант для быстрого тестирования или закрытых сетей. **Не работает с Xray-core** (из-за запрета `allowInsecure`), требует использования клиента на базе **sing-box**.

**Чеклист настройки:**
1. **Генерация сертификата:** Выполните в терминале на сервере:
   ```bash
   openssl req -x509 -nodes -newkey rsa:2048 -keyout hysteria/certs/server.key -out hysteria/certs/server.crt -subj "/CN=bing.com" -days 3650
   ```
2. **Настройка config.yaml:**
   - Закомментируйте весь блок `acme:`.
   - Раскомментируйте блок `tls:` (Вариант Б).
   - Убедитесь, что включен блок `obfs: type: salamander` и задан пароль.
3. **Запуск:** Выполните `docker-compose up -d`.
4. **Подключение клиента (только sing-box!):** Запустите `python generate_uri.py` или соберите ссылку:
   `hysteria2://user:pass@ВАШ_IP:4433/?sni=bing.com&obfs=salamander&obfs-password=ВАШ_ПАРОЛЬ&insecure=1#Hysteria_Self`
   
   > 💡 **Важно про SNI:** Если при генерации сертификата (шаг 1) вы указали не `bing.com`, а другой домен, обязательно замените его в параметре `?sni=` ссылки. Генератор ссылок автоматически подхватит домен, если вы укажете его в `config.yaml` (даже в комментарии рядом с TLS).

### 🔵 Сценарий 3: Мимикрия + ACME на Домен
Классический метод маскировки. Сервер будет притворяться обычным HTTPS-сайтом. Требует наличия собственного доменного имени. **Совместим с Xray-core и sing-box.**

**Чеклист настройки:**
1. **Подготовка:**
   - Направьте ваш домен (A-запись) на IP-адрес сервера.
   - Убедитесь, что порт **80** открыт.
2. **Настройка config.yaml:**
   - Раскомментируйте блок `acme:` (Вариант А).
   - Впишите свой **домен** вместо `"ВАШ_IP_ИЛИ_ДОМЕН"`.
   - Закомментируйте блок обфускации (`obfs:`).
   - Раскомментируйте блок `masquerade:` и укажите сайт для мимикрии (например, bing.com).
3. **Права доступа (только Linux):** Выполните `sudo chown -R 1000:1000 hysteria/data`.
4. **Запуск:** Выполните `docker-compose up -d`. Проверьте логи `docker logs -f hysteria2`, чтобы убедиться, что сертификат на домен получен.
5. **Подключение клиента:** Запустите `python generate_uri.py` или соберите ссылку:
   `hysteria2://user:pass@ВАШ_ДОМЕН:4433/?sni=ВАШ_ДОМЕН#Hysteria_Mimicry`
   *(Обфускация и insecure отключены, так как это чистый QUIC с валидным сертификатом).*

## ⚙️ Управление и Утилиты

**Автоматическая генерация ссылок:**
Если вы настроили `config.yaml`, просто запустите скрипт генератора. Он сам подставит нужные IP, пароли и параметры insecure.
```bash
python generate_uri.py
```

**Команды Docker:**
* Запуск сервера: `docker-compose up -d`
* Остановка сервера: `docker-compose down`
* Просмотр логов: `docker logs -f hysteria2`
* Применение изменений в конфиге: `docker-compose restart`

**Важные нюансы паролей:**
Избегайте использования спецсимволов `@`, `:`, `/`, `#` в паролях пользователей в `config.yaml`, так как они ломают структуру URI-ссылок при импорте в клиент.
